Adapter Design Pattern

适配器模式（Adapter Design Pattern）是一种结构型设计模式。把一个结构的一组方法转换成客户端所期待的另一种接口，从而使原本因接口不匹配而无法一起工作的两个结构能够一起工作。对于这个模式，有一个经常被拿来解释它的例子，就是 USB 转接头充当适配器，把两种不兼容的接口，通过转接变得可以一起工作。

UML Diagram

  • Client 为主调方，依赖 Target 接口;
  • Target 表示目标接口定义;
  • Adaptee 为一组不兼容 Target 接口定义的接口（被适配的结构）;
  • Adaptor 将 Adaptee 转化成一组符合 Target 接口定义的接口;

适配器模式也可以称为 Wrapper 模式，通过构建（内嵌一个接口不匹配的 Adaptee）Wrapper 结构来对 Adaptee 进行能力转换以实现 Target 接口。由于 Client 的业务逻辑依赖的是抽象（Target 接口）而非具体的结构类型，原本无法与 Client 对接的 Adaptee 经过 Adapter 包装也可以与 Client 对接了。

Example

Question：

mac 电脑具备接入 Lightning 插头的能力。
windows 电脑具备接入 USB 插头的能力。
client 希望将 Lightning 插头接入两台电脑，应如何处理？

Code:

 1package main
 2
 3import "fmt"
 4
 5// 主调方依赖的目标接口
 6type computer interface {
 7	insertIntoLightningPort()
 8}
 9
10// 已实现目标接口
11type mac struct{}
12
13func (m *mac) insertIntoLightningPort() {
14	fmt.Println("Lightning connector is plugged into mac machine.")
15}
16
17// 未实现目标接口，待适配
18type windows struct{}
19
20func (w *windows) insertIntoUSBPort() {
21	fmt.Println("USB connector is plugged into windows machine.")
22}
23
24// 适配器
25type windowsAdapter struct {
26	windowMachine *windows
27}
28
29// 适配器通过能力转换（适配），实现目标接口
30func (w *windowsAdapter) insertIntoLightningPort() {
31	fmt.Println("Adapter converts Lightning signal to USB......")
32	w.windowMachine.insertIntoUSBPort()
33}
34
35// 主调方
36type client struct{}
37
38// 主调方的业务逻辑依赖的是目标接口，而非具体的实现
39func (c *client) insertLightningConnectorIntoComputer(com computer) {
40	com.insertIntoLightningPort()
41}
42
43func main() {
44	client := &client{}
45	mac := &mac{}
46
47	client.insertLightningConnectorIntoComputer(mac)
48
49	windowsMachine := &windows{}
50	windowsMachineAdapter := &windowsAdapter{
51		windowMachine: windowsMachine,
52	}
53
54	client.insertLightningConnectorIntoComputer(windowsMachineAdapter)
55}

Output:

Lightning connector is plugged into mac machine.
Adapter converts Lightning signal to USB......
USB connector is plugged into windows machine.

Application Scenarios

  1. 封装有缺陷的接口设计
  2. 统一多个类的接口设计
  3. 替换依赖的外部系统
  4. 兼容老版本接口
  5. 适配不同格式的数据
